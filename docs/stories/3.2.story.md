# Story 3.2 — Yahoo Finance Price Adapter (International ETFs)

**Phase:** 3 — Price Engine
**Status:** Draft
**Effort:** M
**Dependencies:** 1.1 (price_cache table), 3.1 (PriceProvider interface)

## Description

As a developer,
I want a price adapter for Yahoo Finance that fetches quotes for ~20 international ETFs with circuit breaker protection,
so that ETF prices are available for portfolio valuation and rebalancing.

Yahoo Finance has no SLA — the adapter must be resilient with a circuit breaker (3 consecutive failures per ticker → 1 hour cooldown). All ~20 ETFs must be validated during Sprint 1.

[Source: PRD §3 Module 5, Feature F-007 AC2]

## Acceptance Criteria

- [ ] Given ~20 international ETF tickers (QQQM, SCHD, AVUV, etc.), when `fetchPrices()` is called, then each ticker gets an individual request to Yahoo Finance
- [ ] Given a successful response, when processed, then each result includes: `ticker`, `price` in USD, `currency: 'USD'`, `source: 'yahoo'`, `fetched_at`
- [ ] Given 3 consecutive failures for a ticker, when the circuit breaker triggers, then that ticker is skipped for 1 hour with a UI alert surfaced
- [ ] Given the circuit breaker state, when cooldown expires, then the next refresh attempt includes the previously-failed ticker
- [ ] Given a ticker that fails persistently, when manual fallback is needed, then the asset can have `price_source` set to `'manual'` via the UI
- [ ] Error: Given Yahoo Finance returns an error or timeout, when detected, then the error is logged to `price_fetch_log` with provider, ticker, status, and error message
- [ ] Error: Given Yahoo Finance is completely down, when all tickers fail, then existing cached prices remain in `price_cache` with original `fetched_at` timestamps
