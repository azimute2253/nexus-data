# Story 6.1 — CRUD Asset Types

**Phase:** 6 — Asset Management
**Status:** Draft
**Effort:** S
**Dependencies:** 1.1 (schema), 1.2 (RLS), 1.3 (auth), 9.1 (route structure for type pages)

## Description

As a user,
I want to list, create, edit (target %), and reorder my 10 asset types,
so that I can manage my portfolio allocation targets without editing a spreadsheet.

[Source: PRD §3 Module 3, Feature F-003]

## Acceptance Criteria

- [ ] Given the asset types page, when loaded, then all 10 types are listed with their target %, sorted by `sort_order`
- [ ] Given the edit form for a type, when I change target % and save, then the new target is persisted and immediately reflected in rebalancing calculations
- [ ] Given the create form, when I enter a name and target % and save, then a new asset type is created with the next `sort_order`
- [ ] Given the reorder interface, when I drag/reorder types, then `sort_order` is updated in the database
- [ ] Given the list, when displayed, then each type shows: name, target %, current value (BRL), number of groups, number of assets
- [ ] Error: Given a target % that would make total targets exceed 100% across all types, when saving, then a validation warning is shown
