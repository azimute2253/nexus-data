# Story 8.3 — Tab Navigation (Overview | Detalhes | Rebalancear)

**Phase:** 8 — Mobile Responsiveness
**Status:** Draft
**Effort:** S
**Dependencies:** 5.1 (dashboard), 5.5 (calculator), 9.1 (route structure)

## Description

As a mobile user,
I want tab-based navigation at the bottom/top of the dashboard with Overview | Detalhes | Rebalancear tabs,
so that I can quickly switch between viewing my portfolio, drilling into details, and running a rebalance.

[Source: PRD §3 Module 7, Feature F-009 AC4]

## Acceptance Criteria

- [ ] Given the dashboard on mobile, when rendered, then 3 navigation tabs are visible: "Overview", "Detalhes", "Rebalancear"
- [ ] Given the "Overview" tab is active, when displayed, then the dashboard summary (total value, allocation table, chart) is shown
- [ ] Given the "Detalhes" tab is tapped, when navigated, then the asset type drill-down view (L1→L2→L3) is shown
- [ ] Given the "Rebalancear" tab is tapped, when navigated, then the rebalance calculator with input and results is shown
- [ ] Given the current tab, when active, then it has a visual indicator (highlighted/underlined)
- [ ] Given desktop viewport, when rendered, then tabs are still present but positioned horizontally in the header area
- [ ] Given a tab navigation failure (e.g., the target section component fails to load), when the tab is tapped, then an error message "Falha ao carregar seção" is displayed in the content area instead of a blank screen
