# Story 3.3 — Exchange Rate Adapter (USD/BRL, BTC/BRL)

**Phase:** 3 — Price Engine
**Status:** Draft
**Effort:** XS
**Dependencies:** 1.1 (exchange_rates table), 3.1 (PriceProvider interface)

## Description

As a developer,
I want an exchange rate adapter for exchangerate-api.com that fetches USD/BRL and BTC/BRL rates,
so that international assets can be correctly valued in BRL for the portfolio summary.

The adapter has a tight budget (1,500 req/month) so cache TTL is 15 minutes minimum.

[Source: PRD §3 Module 5, Feature F-007 AC3]

## Acceptance Criteria

- [ ] Given a call to `fetchRates()`, when exchangerate-api responds, then USD/BRL and BTC/BRL rates are returned
- [ ] Given a successful response, when processed, then each result includes: `pair` (e.g. 'USD/BRL'), `rate`, `source: 'exchangerate-api'`, `fetched_at`
- [ ] Given the exchange rate response, when processed, then rates are written to the `exchange_rates` table with `fetched_at` timestamp
- [ ] Given the 1,500 req/month budget, when the adapter runs, then it respects minimum 15-minute intervals between requests
- [ ] Given the API key, when the adapter is used, then the key is read from Supabase Edge Function secrets
- [ ] Error: Given exchangerate-api returns an error, when detected, then the last cached rate remains in `exchange_rates` and the error is logged

## Test Requirements

| Test | Input | Expected | Validation |
|------|-------|----------|------------|
| T3.3.1 | Fetch USD/BRL | Rate between 4.0 and 8.0 (reasonable range) | `rate > 4.0 && rate < 8.0` |
| T3.3.2 | Fetch BTC/BRL | Positive rate | `rate > 0` |
| T3.3.3 | API error | Last cached rate unchanged | `exchange_rates` row unchanged |
| T3.3.4 | Second fetch within 15 minutes | No API call made, cached rate returned | Request count = 0, cached rate returned |
