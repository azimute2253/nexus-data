# Story 6.4 — Asset Flags (is_active + manual_override)

**Phase:** 6 — Asset Management
**Status:** Draft
**Effort:** S
**Dependencies:** 6.3 (asset CRUD must exist)

## Description

As a user,
I want to toggle `is_active` and `manual_override` flags on any asset across all 10 types,
so that I can exclude assets from rebalancing without deleting them — replacing the spreadsheet's "Grupo 0" hack and the FII-only "Vou aportar?" flag.

This directly addresses PRD Goal G6 and Creative problem #6 (override centralizado).

[Source: PRD §3 Module 3, Feature F-005 AC3/AC4]

## Acceptance Criteria

- [ ] Given any asset in any of the 10 types, when I toggle `is_active` to false, then the asset is excluded from rebalancing without being deleted from the database
- [ ] Given any asset, when I toggle `manual_override` to true, then the asset is temporarily excluded from rebalancing (equivalent to "Vou aportar? Nao")
- [ ] Given an inactive asset, when the asset list is displayed, then the asset is visually dimmed/grayed with an "Inactive" badge
- [ ] Given an overridden asset, when the asset list is displayed, then the asset shows a "Pausado" badge
- [ ] Given the flags are toggled, when the rebalance algorithm runs, then excluded assets receive 0 shares and their group allocation is redistributed among remaining active assets
- [ ] Given all 10 asset types, when inspected, then both flags are available for every type (not just FIIs) — covers PRD Goal G6
- [ ] Error: Given a group where all assets have `is_active = false` or `manual_override = true`, when rebalancing, then the group's allocation is returned as unallocated remainder
