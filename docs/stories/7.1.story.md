# Story 7.1 — Questionnaire Editor

**Phase:** 7 — Scoring System
**Status:** Draft
**Effort:** M
**Dependencies:** 1.1 (schema — questionnaires table), 9.1 (route for /dashboard/questionnaires)

## Description

As a user,
I want to create and edit scoring questionnaires with Sim/Nao questions and +1/-1 weights,
so that I can customize how assets are scored within their groups.

Different questionnaires exist for different asset types (FIIs, Acoes, ETFs). Each questionnaire has an ordered list of questions with Sim/Nao answers and weights.

[Source: PRD §3 Module 4, Feature F-006 AC1/AC2/AC3]

## Acceptance Criteria

- [ ] Given the questionnaires page, when loaded, then existing questionnaires are listed (FIIs, Acoes, ETFs)
- [ ] Given the create form, when I define a questionnaire with a name, linked asset type, and at least one Sim/Nao question with weight (+1 or -1), then it is saved to the database and appears in the list
- [ ] Given a questionnaire, when I add a question, then the question is saved with: question text (non-empty string), weight (+1 or -1), and sort_order (integer)
- [ ] Given an existing questionnaire, when I edit a question's text, weight, or sort_order, then the updated values are persisted to the database
- [ ] Given the editor with questions in order [A, B, C], when I move question C to position 1, then the sort_order values update so the list displays [C, A, B]
- [ ] Given 3 questionnaire types (FIIs, Acoes, ETFs), when listed, then each shows the linked asset type and the count of questions
- [ ] Error: Given a questionnaire with zero questions, when saving, then a validation error "Questionario deve ter pelo menos 1 pergunta" is shown and the save is blocked
- [ ] Error: Given a question with empty text, when saving, then a validation error is shown and the save is blocked
