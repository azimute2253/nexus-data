# Story 5.1 — Portfolio Summary View & Dashboard Data Layer

**Phase:** 5 — Dashboard & Portfolio View
**Status:** Draft
**Effort:** M
**Dependencies:** 1.1 (schema + portfolio_summary view), 3.4 (price_cache populated), 2.2 (data migrated)

## Description

As a user,
I want the dashboard to display my total portfolio value consolidated in BRL (domestic + international converted via exchange rate),
so that I immediately see how much my portfolio is worth.

This story implements the server-side data fetching in the Astro page: querying the `portfolio_summary` view, computing total value with currency conversion, and passing the data as props to React islands.

[Source: PRD §3 Module 1, Feature F-001 AC1]

## Acceptance Criteria

- [ ] Given the dashboard page loads, when the `portfolio_summary` view is queried, then it returns aggregated value per asset type with current prices from `price_cache`
- [ ] Given international assets priced in USD, when the total is calculated, then USD values are converted to BRL using the latest `exchange_rates` entry
- [ ] Given all data is loaded, when the total is computed, then it reflects `sum(quantity * price * exchange_rate)` across all 131+ active assets
- [ ] Given the Astro page, when it renders, then portfolio data is fetched server-side in the frontmatter and passed as props to React islands
- [ ] Given a price in `price_cache` older than 24 hours, when displayed, then a staleness warning badge is shown with the `fetched_at` timestamp
- [ ] Error: Given `price_cache` has no price for an asset, when the summary is computed, then that asset's value is shown as "N/A" with a visual indicator
