# Story 8.1 — Card-Based Mobile Layout

**Phase:** 8 — Mobile Responsiveness
**Status:** Draft
**Effort:** M
**Dependencies:** 5.2 (allocation table), 6.3 (asset table), 5.1 (dashboard)

## Description

As a mobile user,
I want tables to collapse into card-based layouts on screens < 768px,
so that I can read portfolio data without horizontal scrolling on viewports as narrow as 320px.

The layout transforms: desktop tables become stacked cards on mobile. Value total and top holdings are visible without scroll.

[Source: PRD §3 Module 7, Feature F-009 AC1/AC3]

## Acceptance Criteria

- [ ] Given a screen width < 768px, when the allocation table renders, then it displays as stacked cards instead of a table
- [ ] Given each card, when displayed, then it shows: type name, target %, current %, value, status badge, deviation indicator
- [ ] Given the asset list, when on mobile, then assets display as cards with: ticker, quantity, price, value
- [ ] Given the dashboard, when on mobile, then total portfolio value and top 5 deviations are visible without scrolling
- [ ] Given all mobile layouts, when inspected, then no horizontal scroll is present on any page
- [ ] Given desktop width (>= 768px), when rendered, then standard table layout is used
- [ ] Given a card with missing data (e.g., null price or undefined deviation), when rendered on mobile, then the card displays a "—" placeholder instead of crashing or showing "undefined"
