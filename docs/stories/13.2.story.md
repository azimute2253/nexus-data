# Story 13.2 — Rename "Portfolio" to "Nexus Data"

**Epic:** 13 — Navigation & Branding
**Status:** Draft
**Effort:** S (1 point)
**Dependencies:** 9.2 (layout integration — knows where "Portfolio" references exist)
**PRD Feature:** F-027

## Description

As a user,
I want all visible references to "Portfolio" replaced with "Nexus Data" across the interface,
so that the product has a consistent identity and I always see the correct name.

This includes the sidebar menu in azimute.cc, browser tab title (`document.title`), breadcrumbs, headers, and any other user-visible text.

[Source: PRD-v2 §3 Module 14, Feature F-027; Creative V2 §Menu lateral]

## Acceptance Criteria

- [ ] Given the azimute.cc sidebar menu, when rendered, then the portfolio link displays "Nexus Data" instead of "Portfolio"
- [ ] Given any Nexus Data page, when the browser tab is inspected, then `document.title` includes "Nexus Data"
- [ ] Given breadcrumbs and page headers, when displayed, then they show "Nexus Data" instead of "Portfolio"
- [ ] Given a full-text search of the codebase for user-visible "Portfolio" strings, when performed, then 0 occurrences are found in UI-facing text (code comments and variable names are exempt)
- [ ] Given internal URLs, when inspected, then the URL slug can remain `/dashboard/portfolio` (technical decision) — only the visible label must be "Nexus Data"
