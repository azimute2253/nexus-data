# Story 12.1 — Wallet Selector Component

**Epic:** 12 — Wallet UX
**Status:** Ready for Review
**Effort:** S (2 points)
**Dependencies:** 11.3 (wallet data access layer)
**PRD Feature:** F-023

## Description

As a user,
I want a wallet selector dropdown in the app header to switch between my wallets,
so that I can quickly alternate between different portfolios without losing my place.

When a user has a single wallet, the selector displays as static text (no dropdown). When multiple wallets exist, a dropdown allows switching. The active wallet is persisted in `localStorage`.

[Source: PRD-v2 §3 Module 12, Feature F-023; ADR-012 (Active Wallet Persistence)]

## Acceptance Criteria

- [ ] Given a user with multiple wallets, when the header renders, then a dropdown lists all wallets ordered by `created_at` ascending
- [ ] Given a user with exactly one wallet, when the header renders, then the wallet name displays as static text (no dropdown trigger)
- [ ] Given the wallet selector, when a different wallet is selected, then all dashboard data refreshes to show the selected wallet's data within < 500ms perceived latency
- [ ] Given a wallet switch, when completed, then the active `wallet_id` is saved to `localStorage` key `nexus_active_wallet_id`
- [ ] Given a page reload, when the app initializes, then it reads `nexus_active_wallet_id` from `localStorage` and selects that wallet (or falls back to first wallet if invalid/missing)
- [ ] Given the selector, when visible, then a "Nova carteira" button/link is accessible to create a new wallet
