# Story 14.2 — Dual Weight UI (Mode Switch + Manual Input + Questionnaire Integration)

**Epic:** 14 — Dual Weight System
**Status:** Ready for Review
**Effort:** M (3 points)
**Dependencies:** 14.1 (schema + data layer), 7.2 (existing ScoringModal component)
**PRD Feature:** F-028

## Description

As a user,
I want to see a weight mode switch on each asset (Manual / Questionário) with the appropriate input for each mode,
so that I can choose how to assign weight to each asset — either by entering a manual note or by answering the questionnaire.

When "Nota manual" is selected, a numeric input (-10 to +11) is shown. When "Questionário" is selected, the existing ScoringModal (from V1 Story 7.2) is shown. The current calculated weight is always displayed regardless of mode.

[Source: PRD-v2 §3 Module 15, Feature F-028; Creative V2 §Sistema de Peso por Ativo]

## Acceptance Criteria

- [x] Given an asset detail/edit panel, when rendered, then a mode switch is visible with two options: "Nota manual" and "Questionário" (radio buttons or toggle)
- [x] Given `weight_mode = 'manual'`, when the panel renders, then a numeric input for the manual weight (-10 to +11) is shown and the questionnaire section is hidden
- [x] Given `weight_mode = 'questionnaire'`, when the panel renders, then the ScoringModal (existing V1 component) is accessible and the manual input is hidden
- [x] Given the manual input, when a value outside -10 to +11 is entered, then a validation error is shown
- [x] Given the panel, when the mode is switched, then the UI immediately updates to show the correct input for the new mode (no save required to see the change)
- [x] Given the panel, when "Salvar" is clicked, then both `weight_mode` and the relevant weight value are persisted to the database
- [x] Given any mode, when the panel is displayed, then the current calculated weight (normalized percentage within the group) is shown as a read-only indicator
- [x] Given a mode switch, when saved, then the previous mode's data is preserved in the DB (manual_weight is not cleared when switching to questionnaire, and vice versa)
