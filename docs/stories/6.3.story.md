# Story 6.3 — CRUD Individual Assets

**Phase:** 6 — Asset Management
**Status:** Draft
**Effort:** M
**Dependencies:** 6.2 (groups must exist), 3.4 (prices for display), 1.1 (schema)

## Description

As a user,
I want to add, edit, and view individual assets with their ticker, sector, quantity, group, and price source,
so that I can manage all 131+ positions in one place instead of editing 3 separate spreadsheet tabs.

[Source: PRD §3 Module 3, Feature F-005 AC1/AC2/AC5/AC6]

## Acceptance Criteria

- [ ] Given a selected group, when assets are loaded, then all assets in that group are listed with: ticker, sector, quantity, current price, value (BRL)
- [ ] Given the add form, when I fill in ticker, sector, quantity, and select a price source (brapi/yahoo/manual/crypto), then a new asset is created in the database
- [ ] Given an existing asset, when I edit any field (ticker, sector, quantity, group, price_source), then the change is persisted
- [ ] Given 131+ assets, when the list renders, then there is no perceptible performance degradation (< 200ms render)
- [ ] Given the price source selector, when displayed, then options are: brapi, yahoo, manual, crypto
- [ ] Given the asset list, when displayed, then each row shows: ticker, sector, quantity, price, value (BRL), is_active status, manual_override status
- [ ] Error: Given a duplicate ticker in the same group, when creating, then a validation error is shown
- [ ] Error: Given an empty ticker field, when saving, then validation prevents the save
