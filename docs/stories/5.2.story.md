# Story 5.2 — Asset Type Allocation Table

**Phase:** 5 — Dashboard & Portfolio View
**Status:** Ready for Review
**Effort:** S
**Dependencies:** 5.1 (dashboard data layer must provide portfolio data)

## Description

As a user,
I want a table on the dashboard showing all 10 asset classes with target %, current %, value (BRL), and status (overweight/underweight),
so that I can see at a glance which classes need attention.

[Source: PRD §3 Module 1, Feature F-001 AC2]

## Acceptance Criteria

- [x] Given the dashboard renders, when the allocation table is displayed, then it lists all 10 asset types in sort_order
- [x] Given each row, when rendered, then it shows: name, target %, current %, value (BRL formatted), status badge
- [x] Given current % > target %, when the status is computed, then the badge shows "Overweight" in red
- [x] Given current % < target %, when the status is computed, then the badge shows "Underweight" in yellow/orange
- [x] Given current % ≈ target % (within 2pp), when the status is computed, then the badge shows "Aligned" in green
- [x] Given the table on desktop, when displayed, then it renders as a standard HTML table with sortable columns
- [x] Given the table on mobile (< 768px), when displayed, then it renders as card-based layout (Story 8.1)
- [x] Error: Given portfolio data fails to load (network error or empty response), when the table renders, then it shows an error state with message "Falha ao carregar alocação" and a retry button
