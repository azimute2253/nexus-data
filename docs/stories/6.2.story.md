# Story 6.2 — CRUD Asset Groups

**Phase:** 6 — Asset Management
**Status:** Draft
**Effort:** S
**Dependencies:** 6.1 (asset types must exist), 1.1 (schema)

## Description

As a user,
I want to list, create, edit, and manage groups within each asset type,
so that I can organize my assets into subgroups with their own target percentages and scoring methods.

[Source: PRD §3 Module 3, Feature F-004]

## Acceptance Criteria

- [ ] Given a selected asset type, when groups are loaded, then all groups for that type are listed with target % and scoring method
- [ ] Given the create form, when I enter a group name and target %, then a new group is created under the selected type
- [ ] Given the edit form for a group, when I change target % or scoring method, then the changes are persisted
- [ ] Given the group targets within a type, when any group is saved, then a validation checks that sum of group target % = 100%
- [ ] Given the group list, when displayed, then each group shows: name, target %, number of assets, scoring method
- [ ] Error: Given group targets that don't sum to 100%, when saving, then a validation error is shown listing the current sum
