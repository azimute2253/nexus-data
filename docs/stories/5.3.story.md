# Story 5.3 — Allocation Pizza Chart (Actual vs Target)

**Phase:** 5 — Dashboard & Portfolio View
**Status:** Draft
**Effort:** S
**Dependencies:** 5.1 (dashboard data layer)

## Description

As a user,
I want a side-by-side pie chart comparing my actual allocation vs target allocation across the 10 asset types,
so that I can visually see where my portfolio deviates from the plan.

Uses Recharts for the pie chart implementation within a React island.

[Source: PRD §3 Module 1, Feature F-001 AC3]

## Acceptance Criteria

- [ ] Given the dashboard renders, when the chart component loads, then two pie charts are displayed side-by-side: "Atual" and "Target"
- [ ] Given the "Atual" chart, when rendered, then each slice represents one asset type's current % of total portfolio
- [ ] Given the "Target" chart, when rendered, then each slice represents one asset type's target %
- [ ] Given both charts, when compared visually, then FIIs slice in "Atual" (65%) is at least 4x larger than in "Target" (15%), with distinct color per asset type
- [ ] Given mobile viewport (< 768px), when rendered, then charts stack vertically instead of side-by-side
- [ ] Given the chart, when a slice is hovered, then a tooltip shows: type name, percentage, value (BRL)
- [ ] Error: Given portfolio data is empty or unavailable, when the chart component renders, then it displays a placeholder with message "Sem dados para exibir" instead of an empty/broken chart
