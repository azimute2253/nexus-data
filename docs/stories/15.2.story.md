# Story 15.2 — Tab Aportes Content (Rebalance Calculator + Contribution History)

**Epic:** 15 — Tab Content
**Status:** Draft
**Effort:** M (3 points)
**Dependencies:** 13.1 (tab navigation), 11.3 (wallet data layer), 5.5 (existing rebalance calculator UI), 4.4 (rebalance orchestrator)
**PRD Feature:** F-030

## Description

As a user,
I want the Aportes tab to provide a contribution calculator (pre-filled with my last value) and a history of past contributions,
so that I can quickly calculate what to buy on my next investment and track my past contributions.

This story extends the existing RebalanceCalculator (Story 5.5) by adding default value logic, contribution history, and wallet-scoped data.

[Source: PRD-v2 §3 Module 16, Feature F-030; Creative V2 §Página: Aportes]

## Acceptance Criteria

- [ ] Given the Aportes tab, when rendered, then it shows: contribution input field, "Calcular" button, result area, and contribution history section
- [ ] Given the input field, when rendered for a wallet with past contributions, then it is pre-filled with the value of the last registered contribution for the active wallet
- [ ] Given the input field, when rendered for a wallet with no past contributions, then it is pre-filled with R$ 12.000 (default value)
- [ ] Given a calculation result, when displayed, then it shows hierarchical L1→L2→L3 breakdown: per class, per group, and per asset with ticker, shares to buy, estimated value, and weight mode indicator (manual/questionário)
- [ ] Given the contribution history section, when the wallet has contributions, then a chronological reverse list shows date, value, and distribution snapshot for each contribution
- [ ] Given the contribution history, when the wallet has no contributions, then it displays "Nenhum aporte registrado ainda"
- [ ] Given the Aportes tab data, when queried, then all results are filtered by the active `wallet_id`
