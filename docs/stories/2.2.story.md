# Story 2.2 — Migration Script (Sheets to Supabase)

**Phase:** 2 — Data Migration
**Status:** Draft
**Effort:** M
**Dependencies:** 1.1 (schema), 1.2 (RLS), 2.1 (parser) | [Ref: ADR-008 (Data Migration Strategy), ADR-007 (Authentication/RLS), ADR-002 (Database)]

## Description

As a developer,
I want a migration script that inserts all parsed portfolio data into Supabase in the correct FK dependency order,
so that the entire portfolio (10 types, ~15 groups, 131+ assets, 3 questionnaires, all scores) is migrated in one run.

The script must be idempotent (UPSERT) so it can be re-run safely. It must authenticate with the correct `user_id` for RLS compliance.

[Source: PRD §3 Module 8, Feature F-010 AC3/AC4]

## Acceptance Criteria

- [ ] Given parsed data from all 6 tabs, when the migration script runs, then data is inserted in FK order: `asset_types` → `questionnaires` → `asset_groups` → `assets` → `asset_scores`
- [ ] Given 10 asset types from the spreadsheet, when migrated, then all 10 exist in `asset_types` with `name`, `target_pct`, and `sort_order` matching the spreadsheet values exactly
- [ ] Given ~15 groups, when migrated, then each group has a valid `type_id` FK referencing an existing `asset_types` row, `target_pct` matching the spreadsheet value, and `scoring_method` matching the questionnaire type for that group
- [ ] Given 131+ assets, when migrated, then each has: `ticker` matching the spreadsheet value (uppercase, trimmed, no `.SA` suffix), `sector`, `quantity` as integer, `group_id` FK referencing an existing `asset_groups` row, `price_source`, `is_active`, and `manual_override`
- [ ] Given scoring data, when migrated, then each asset has scores linked to the matching questionnaire type (FIIs/Acoes/ETFs) with weights preserving sign and magnitude from the spreadsheet (including negatives such as -10)
- [ ] Given the script is run twice, when completed, then no duplicate rows exist (UPSERT on conflict)
- [ ] Given all inserted data, when inspected, then all rows have `user_id` matching Luis's auth.uid()
- [ ] Error: Given a missing FK reference (e.g., asset referencing non-existent group), when the script runs, then it fails with a clear error message before proceeding
