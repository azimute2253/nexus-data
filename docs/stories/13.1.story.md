# Story 13.1 — Tab Navigation: Dashboard / Aportes / Ativos

**Epic:** 13 — Navigation & Branding
**Status:** Draft
**Effort:** M (3 points)
**Dependencies:** 8.3 (existing TabNavigation component to be updated), 9.1 (route structure)
**PRD Feature:** F-026

## Description

As a user,
I want the main navigation tabs renamed and reconfigured as Dashboard / Aportes / Ativos,
so that the interface better reflects the V2 functionality and I can navigate between viewing my portfolio, managing contributions, and configuring assets.

This updates the existing `TabNavigation.tsx` component (from Story 8.3) with new tab labels and content areas. Tab state is persisted in the URL via query parameter for deep-linking.

[Source: PRD-v2 §3 Module 14, Feature F-026; ADR-013 (Tab Routing Strategy)]

## Acceptance Criteria

- [ ] Given the Nexus Data page, when rendered, then 3 tabs are visible: "Dashboard", "Aportes", "Ativos"
- [ ] Given the "Dashboard" tab is active, when displayed, then the dashboard content (portfolio overview) renders
- [ ] Given the "Aportes" tab is tapped, when navigated, then the contributions/rebalance content renders
- [ ] Given the "Ativos" tab is tapped, when navigated, then the asset hierarchy/configuration content renders
- [ ] Given the active tab, when the URL is inspected, then the query parameter reflects the active tab (e.g., `?tab=aportes` or `?tab=ativos`; Dashboard is default with no param)
- [ ] Given a URL with `?tab=ativos`, when the page is loaded directly, then the Ativos tab is active (deep-linking support)
- [ ] Given tab switching, when transitioning, then data is cached and no re-fetch occurs (instant transition)
- [ ] Given mobile viewport, when rendered, then tabs use horizontal layout with thumb-friendly touch targets (min 44px)
