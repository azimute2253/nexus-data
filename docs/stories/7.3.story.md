# Story 7.3 — Score Normalization Logic

**Phase:** 7 — Scoring System
**Status:** Draft
**Effort:** S
**Dependencies:** 7.2 (scores must be saved for normalization), 4.3 (L3 consumes normalized scores)

## Description

As a developer,
I want the score normalization function that converts raw scores into ideal % per asset within a group,
so that the L3 rebalancing distribution uses correct proportions based on questionnaire results.

Score normalization handles negative scores by shifting the minimum to zero before computing proportions. This ensures all assets get non-negative allocation weights.

[Source: PRD §3 Module 4, Feature F-006 AC5]

## Acceptance Criteria

- [ ] Given assets in a group with scores [8, 5, 3, -2], when normalized, then each gets a proportional % of the group's allocation
- [ ] Given the normalization, when a negative score exists, then the minimum is shifted to zero: [8, 5, 3, -2] → [10, 7, 5, 0] → percentages: [45.5%, 31.8%, 22.7%, 0%]
- [ ] Given all scores in a group are identical (e.g., all 5), when normalized, then each asset gets equal % (1/N)
- [ ] Given a single asset in a group, when normalized, then it gets 100%
- [ ] Given the `normalizeScores()` function in `src/lib/nexus/rebalance.ts`, when called, then it is a pure function with no side effects, no database calls, and no imports beyond TypeScript standard library
- [ ] Error: Given all assets in a group have score 0 (after shift), when normalized, then equal distribution (1/N) is used as fallback — e.g., 3 assets each get 33.33%
- [ ] Error: Given an empty scores array [], when `normalizeScores([])` is called, then it returns an empty array [] without throwing
