# Story 15.3 — Tab Ativos Content (Hierarchical Tree + CRUD + Dual Weight)

**Epic:** 15 — Tab Content
**Status:** Draft
**Effort:** L (5 points)
**Dependencies:** 13.1 (tab navigation), 11.3 (wallet data layer), 14.2 (dual weight UI), 6.1 (asset type CRUD), 6.2 (group CRUD), 6.3 (asset CRUD)
**PRD Feature:** F-031

## Description

As a user,
I want the Ativos tab to display my portfolio as a hierarchical tree (Classes → Groups → Assets) with inline CRUD and access to the dual weight panel,
so that I can configure my entire portfolio structure and weights from a single, intuitive view.

This is the most complex tab, integrating existing CRUD components (Stories 6.1-6.3) into a tree view with the new dual weight panel (Story 14.2).

[Source: PRD-v2 §3 Module 16, Feature F-031; Creative V2 §Página: Ativos]

## Acceptance Criteria

- [ ] Given the Ativos tab, when rendered with data, then a hierarchical tree displays: Asset Classes → Groups → Assets with clear visual indentation
- [ ] Given each level of the tree, when interacted with, then inline create, edit, and delete actions are available (no navigation to another screen)
- [ ] Given an asset in the tree, when displayed, then it shows the current weight with mode indicator: "peso: 7 (questionário)" or "peso: 5 (manual)"
- [ ] Given the [editar] action on an asset, when clicked, then the DualWeightPanel (Story 14.2) opens as a modal or side panel
- [ ] Given all tree data, when queried, then it is filtered by the active `wallet_id`
- [ ] Given asset class creation, when the form is submitted, then it accepts: name, target % (numeric), and sort order
- [ ] Given group creation, when the form is submitted, then it accepts: name, target % within the class, and default scoring method (manual/questionário)
- [ ] Given asset creation, when the form is submitted, then it accepts: ticker, name (optional), sector (optional), quantity, price source, is_active, manual_override, whole_shares
- [ ] Given the sum of asset class target %, when it differs from 100%, then a warning is displayed (not a blocking error)
- [ ] Given the sum of group target % within a class, when it differs from 100% by more than 1 percentage point, then a warning is displayed
- [ ] Given the tree on mobile, when rendered, then the hierarchy is readable with collapsible levels and appropriate touch targets
