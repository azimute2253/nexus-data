# Story 4.2 — L2 Distribution (Group-Level Rebalancing)

**Phase:** 4 — Rebalancing Algorithm
**Status:** Draft
**Effort:** S
**Dependencies:** 4.1 (L1 must produce per-type amounts for L2 input), ADR-004 (Rebalancing Algorithm Architecture)

## Description

As a user,
I want the L2 cascade to distribute each type's allocated amount among its groups based on group target percentages,
so that within each asset class the money flows to the correct subgroups.

L2 takes the output of L1 (amount per type) and distributes within each type among its groups according to each group's `target_pct`. The sum of group targets within a type must equal 100%.

[Source: PRD §3 Module 2, Feature F-002 AC4]

## Acceptance Criteria

- [ ] Given L1 allocates R$4,428 to "Acoes BR" with 4 groups (60%, 20%, 10%, 10%), when `distributeL2()` runs, then groups receive R$2,656.80, R$885.60, R$442.80, R$442.80 respectively
- [ ] Given a type with a single group at 100%, when L2 distributes, then the full type amount goes to that group
- [ ] Given a type that received R$0 from L1, when L2 processes it, then all its groups receive R$0
- [ ] Given the function, when inspected, then it is a pure function with no side effects
- [ ] Error: Given the sum of group targets within a type does not equal 100%, when `distributeL2()` is called, then it throws a descriptive validation error
