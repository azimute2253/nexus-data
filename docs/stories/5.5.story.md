# Story 5.5 — Rebalancing Calculator UI

**Phase:** 5 — Dashboard & Portfolio View
**Status:** Draft
**Effort:** M
**Dependencies:** 4.4 (rebalance algorithm), 5.1 (dashboard data layer), 3.4 (prices available)

## Description

As a user,
I want an aporte input field and "Calcular" button that runs the rebalance algorithm and displays a hierarchical result (L1→L2→L3) with expand/collapse,
so that I know exactly what to buy with my monthly contribution.

This is the core interaction of the app — the "2 toques" flow from the PRD.

[Source: PRD §3 Module 2, Feature F-002 AC1/AC2/AC6]

## Acceptance Criteria

- [ ] Given the calculator component, when rendered, then it shows a numeric input field with default value R$12,000 and a "Calcular" button
- [ ] Given the user enters R$12,000 and clicks "Calcular", when the algorithm runs, then the result is displayed hierarchically: L1 types → L2 groups → L3 assets
- [ ] Given the result, when displayed, then each level is expandable/collapsible (L1 expanded by default, L2/L3 collapsed)
- [ ] Given the L3 result, when displayed per asset, then it shows: ticker, shares to buy, estimated cost in BRL (e.g., "VALE3 — 2 cotas — R$ 154,26")
- [ ] Given the calculation, when completed, then execution time is < 500ms for 131+ assets
- [ ] Given the input field, when the user changes the value and clicks "Calcular" again, then the result updates to reflect the new aporte amount within 500ms
- [ ] Error: Given an invalid input (negative, non-numeric), when entered, then the "Calcular" button is disabled with a validation message
