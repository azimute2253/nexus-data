# Story 9.3 — Feature Flag & Staleness Fallback UI

**Phase:** 9 — Integration & Routes
**Status:** Draft
**Effort:** S
**Dependencies:** 9.1 (routes), 3.4 (price cache for staleness detection)

## Description

As a developer,
I want a feature flag (`PUBLIC_NEXUS_ENABLED`) gating portfolio routes and a consistent staleness/fallback UI when prices are unavailable,
so that incomplete features don't reach production and the app gracefully handles missing data.

[Source: PRD §3 Module 5 F-007 AC6, ADR-001 Obligation 4]

## Acceptance Criteria

- [ ] Given `PUBLIC_NEXUS_ENABLED=false`, when any `/dashboard/portfolio/*` route is accessed, then a "Coming Soon" page is shown instead of the portfolio
- [ ] Given `PUBLIC_NEXUS_ENABLED=true`, when routes are accessed, then the full portfolio UI renders normally
- [ ] Given a price in `price_cache` with `fetched_at` older than 24 hours, when displayed in any component, then a warning badge shows "Stale" + the timestamp
- [ ] Given an API failure indicator in `price_cache`, when the dashboard loads, then assets with failed prices show "Last: R$ X.XX (2h ago)" with a yellow warning icon
- [ ] Given all prices are fresh (< 5 min old), when displayed, then no staleness indicators are shown
- [ ] Given the `PUBLIC_NEXUS_ENABLED` env var is missing or undefined, when any `/dashboard/portfolio/*` route is accessed, then the system defaults to `false` and shows the "Coming Soon" page
