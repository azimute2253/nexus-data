# Story 15.1 — Tab Dashboard Content (Portfolio Overview)

**Epic:** 15 — Tab Content
**Status:** Ready for Review
**Effort:** M (3 points)
**Dependencies:** 13.1 (tab navigation), 11.3 (wallet data layer), 5.1 (existing portfolio summary), 5.2 (allocation table), 5.3 (allocation chart), 5.4 (deviation indicators), 3.5 (price refresh)
**PRD Feature:** F-029

## Description

As a user,
I want the Dashboard tab to display a complete portfolio overview — total value, allocation table, chart, deviation indicators, and price refresh — filtered by my active wallet,
so that I get an instant snapshot of my portfolio health without navigating to multiple screens.

This story consolidates and extends Story 5.1 (portfolio summary) by integrating it into the new tab structure and filtering all data by the active `wallet_id`.

[Source: PRD-v2 §3 Module 16, Feature F-029; Creative V2 §Página: Dashboard]

## Acceptance Criteria

- [x] Given the Dashboard tab, when rendered with data, then it shows: total consolidated value (BRL), asset class allocation table, allocation chart (current vs. target), deviation indicators, "Atualizar preços" button, and last refresh timestamp
- [x] Given all data on the Dashboard, when inspected, then everything is filtered by the active `wallet_id`
- [x] Given international assets, when the total value is calculated, then they are converted via USD/BRL exchange rate from the `exchange_rates` table
- [x] Given the allocation chart, when rendered, then it displays at minimum a pie/donut chart showing current vs. target allocation per asset class
- [x] Given a Dashboard with no data (empty wallet), when rendered, then it shows an empty state: "Sua carteira está vazia. Comece adicionando suas classes de ativo na aba Ativos." with a link to the Ativos tab
- [x] Given a wallet switch via the selector, when the Dashboard tab is active, then all data refreshes to show the new wallet's data
