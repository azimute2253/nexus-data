# Story 3.4 — Price Cron Edge Function

**Phase:** 3 — Price Engine
**Status:** Draft
**Effort:** M
**Dependencies:** 3.1 (brapi adapter), 3.2 (yahoo adapter), 3.3 (exchange adapter), 1.1 (price_cache table)

## Description

As a developer,
I want a Supabase Edge Function that runs on a cron schedule to refresh all prices,
so that the `price_cache` and `exchange_rates` tables always have fresh data during market hours.

The cron runs every 5 minutes during B3 pregao (10:00-17:00 BRT, Mon-Fri) and every 1 hour outside. It orchestrates all 3 providers sequentially, writes results to the cache, and logs all fetch attempts.

[Source: PRD §3 Module 5, Feature F-007 AC4/AC7]

## Acceptance Criteria

- [ ] Given pregao hours (10:00-17:00 BRT, Mon-Fri), when the cron fires, then all 3 providers are called: brapi (6 batches), Yahoo (~20 requests), exchange (1 request)
- [ ] Given outside pregao hours, when the cron fires, then the interval is 1 hour instead of 5 minutes
- [ ] Given weekends or B3 holidays, when the cron would fire, then no API calls are made (skip)
- [ ] Given successful price fetches, when written to `price_cache`, then UPSERT updates existing rows or inserts new ones
- [ ] Given the Edge Function, when it executes, then it uses `SUPABASE_SERVICE_ROLE_KEY` to bypass RLS for writes
- [ ] Given the cron completes a full cycle, when measured, then total execution time is < 10 seconds
- [ ] Given all fetch attempts, when completed, then each is logged to `price_fetch_log` with: provider, tickers, status, response_time_ms, error_message, created_at
- [ ] Error: Given one provider fails (e.g., Yahoo down), when the cron runs, then the other 2 providers still execute and write their results
