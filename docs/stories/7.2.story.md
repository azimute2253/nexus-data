# Story 7.2 — Scoring Modal

**Phase:** 7 — Scoring System
**Status:** Draft
**Effort:** M
**Dependencies:** 7.1 (questionnaires must exist), 6.3 (assets must exist for scoring)

## Description

As a user,
I want a scoring modal that lets me select an asset, answer Sim/Nao questions from the linked questionnaire, and see the score calculated automatically,
so that I can rate assets and the system adjusts their ideal allocation within the group.

[Source: PRD §3 Module 4, Feature F-006 AC4/AC5/AC6/AC7]

## Acceptance Criteria

- [ ] Given an asset in the asset list, when I click "Pontuar", then a modal opens with the questionnaire linked to that asset's type
- [ ] Given the modal, when I answer each question Sim/Nao, then the score is calculated in real-time as sum(answer * weight) — where Sim=1, Nao=0
- [ ] Given all questions answered, when I save, then the answers are persisted in `asset_scores` and the total score is stored
- [ ] Given a saved score, when I reopen the scoring modal for the same asset, then previous answers are pre-filled
- [ ] Given a negative total score (e.g., XPLG11 at -10), when displayed, then the score shows as "-10" with no formatting errors
- [ ] Given the score is saved, when the rebalancing algorithm runs next, then the new score is reflected in the asset's ideal % within its group
- [ ] Error: Given the modal is closed without saving, when reopened, then unsaved answers are discarded and the modal shows the last saved state (no partial state persisted)
- [ ] Error: Given the scoring modal is opened for an asset whose type has no linked questionnaire, when opened, then an error message "Nenhum questionario vinculado a este tipo de ativo" is shown
