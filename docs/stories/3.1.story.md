# Story 3.1 — brapi.dev Price Adapter (B3 Stocks + FIIs)

**Phase:** 3 — Price Engine
**Status:** Ready for Review
**Effort:** S
**Dependencies:** 1.1 (price_cache table), 1.2 (RLS)

## Description

As a developer,
I want a price adapter for brapi.dev that fetches quotes for ~111 B3 tickers in batches of 20,
so that all Brazilian stocks and FIIs have reliable, up-to-date prices.

The adapter implements the `PriceProvider` interface and handles batch requests, error responses, and rate limit awareness. It writes results to the `price_cache` table.

[Source: PRD §3 Module 5, Feature F-007 AC1]

## Acceptance Criteria

- [x] Given ~111 B3 tickers, when `fetchPrices()` is called, then it makes ceil(111/20) = 6 batch requests to brapi.dev
- [x] Given a batch of 20 tickers, when brapi responds with prices, then each ticker's price is returned in BRL matching the `regularMarketPrice` field from the API response
- [x] Given the adapter, when it implements `PriceProvider` interface, then it has: `fetchPrices(tickers: string[]): Promise<PriceResult[]>`
- [x] Given a successful response, when processed, then each result includes: `ticker`, `price`, `currency: 'BRL'`, `source: 'brapi'`, `fetched_at`
- [x] Given the brapi API key, when the adapter is used, then the key is read from Supabase Edge Function secrets (never client-side)
- [x] Error: Given brapi returns HTTP 429 (rate limited), when detected, then the adapter logs the error and returns partial results for successful tickers
- [x] Error: Given brapi returns an error for a specific ticker, when detected, then that ticker is logged to `price_fetch_log` and skipped
