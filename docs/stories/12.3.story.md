# Story 12.3 — Onboarding Flow (First Wallet)

**Epic:** 12 — Wallet UX
**Status:** Ready for Review
**Effort:** M (3 points)
**Dependencies:** 11.3 (wallet data access layer), 13.1 (tab navigation must exist for post-onboarding redirect)
**PRD Feature:** F-025

## Description

As a new user,
I want to be guided through creating my first wallet when I access Nexus Data for the first time,
so that I have a clear starting point and don't see an empty, confusing interface.

The onboarding is detected server-side (Astro frontmatter queries wallets) and renders conditionally on the same URL. After creating the first wallet, the user sees an empty Dashboard with guidance to start in the Ativos tab.

[Source: PRD-v2 §3 Module 12, Feature F-025; ADR-016 (Onboarding Flow)]

## Acceptance Criteria

- [x] Given a user with 0 wallets, when they access the Nexus Data page, then the onboarding screen renders instead of the Dashboard (server-side detection, no flash of wrong content)
- [x] Given the onboarding screen, when displayed, then it shows: welcome message ("Bem-vindo ao Nexus Data"), explanation text, and a wallet name input field with a "Criar minha carteira" button
- [x] Given the onboarding form, when a valid name is submitted, then a wallet is created, the `wallet_id` is saved to `localStorage`, and the page reloads to show the Dashboard
- [x] Given the post-onboarding Dashboard, when loaded with an empty wallet, then it shows an empty state: "Sua carteira está vazia. Comece adicionando suas classes de ativo na aba Ativos." with a link/button to navigate to the Ativos tab
- [x] Given a returning user with wallets, when they access Nexus Data, then the onboarding screen does NOT appear (goes straight to Dashboard)
- [x] Given the onboarding detection, when the Supabase query fails (service error), then a generic error page renders (NOT the onboarding screen)
- [x] Given the onboarding on mobile, when displayed, then it is fully responsive with no horizontal scroll
