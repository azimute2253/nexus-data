# Story 2.1 — CSV Parser for Google Sheets Export

**Phase:** 2 — Data Migration
**Status:** Ready for Review
**Effort:** S
**Dependencies:** 1.1 (schema types needed for transformation targets) | [Ref: ADR-008 (Data Migration Strategy), ADR-002 (Database)]

## Description

As a developer,
I want a CSV parser that reads the 6 Google Sheets tabs and transforms Brazilian-format data into structured JSON matching the Supabase schema,
so that the migration script can reliably insert all portfolio data.

The existing `scripts/fetch-sheet.mjs` handles CSV export. This story adds the parsing and transformation layer using PapaParse with Brazilian format handling (R$ 1.234,56) and data cleaning rules.

[Source: PRD §3 Module 8, Feature F-010 AC1/AC2]

## Acceptance Criteria

- [x] Given a CSV export from the "Distribuicao de aporte" tab, when parsed, then 10 `asset_types` records are produced, each with `name` and `target_pct` matching the spreadsheet values
- [x] Given Brazilian number format "R$ 1.234,56", when parsed, then the value is converted to the numeric value `1234.56` (strip `R$`, remove `.` thousands separator, replace `,` decimal separator with `.`)
- [x] Given a CSV cell containing `#N/A` or `#REF!`, when parsed, then the value becomes `null` and is flagged for manual review
- [x] Given ticker "VALE3.SA", when normalized, then it becomes "VALE3" (strip `.SA` suffix, uppercase, trim)
- [x] Given the "Balanceamentos" tab, when parsed, then questionnaire structure (questions + weights) and per-asset scores (including negatives like -10) are extracted
- [x] Given empty cells in quantity fields, when parsed, then the default value is `0`
- [x] Error: Given a CSV with unexpected column headers, when parsed, then a descriptive error is thrown listing expected vs actual columns
