# Story 5.4 — Deviation Indicators (Visual Overweight/Underweight)

**Phase:** 5 — Dashboard & Portfolio View
**Status:** Done
**Effort:** S
**Dependencies:** 5.2 (allocation table with status data)

## Description

As a user,
I want visual deviation indicators (colored bars proportional to deviation) for each asset class,
so that I can instantly see which classes have significant deviations (e.g., FIIs +50pp overweight).

[Source: PRD §3 Module 1, Feature F-001 AC4]

## Acceptance Criteria

- [x] Given an asset type with significant deviation (e.g., FIIs 65% vs 15%), when displayed, then a red bar indicator is shown proportional to the deviation magnitude
- [x] Given a deviation > 10pp overweight, when rendered, then the bar is red
- [x] Given a deviation > 10pp underweight, when rendered, then the bar is orange/yellow
- [x] Given a deviation within 2pp, when rendered, then no deviation bar is shown (aligned)
- [x] Given the deviation bar, when rendered, then its width is proportional to the deviation magnitude (e.g., +50pp bar is 10x the width of a +5pp bar, scaled linearly to a max-width of 100%)
- [x] Given the mobile viewport, when rendered, then deviation indicators are visible without horizontal scroll (F-001 AC5)
- [x] Error: Given an asset type with missing or null target_pct, when the deviation is computed, then no bar is rendered and the cell displays "—" (dash)
