# Story 9.1 — Route Structure & Hierarchical Navigation

**Phase:** 9 — Integration & Routes
**Status:** Draft
**Effort:** M
**Dependencies:** 1.3 (auth protecting /dashboard/*), 1.1 (schema for dynamic routes)

## Description

As a user,
I want the complete route structure with hierarchical navigation (L1→L2→L3) and breadcrumbs,
so that I can navigate between portfolio overview, asset types, groups, and individual assets with deep-linkable URLs.

[Source: PRD §3 Module 9, Feature F-011 AC1/AC2/AC3]

## Acceptance Criteria

- [ ] Given the route `/dashboard/portfolio`, when accessed, then the dashboard central (L1) renders with total portfolio value, allocation table, and deviation chart visible
- [ ] Given the route `/dashboard/assets/[type]`, when accessed with a valid type slug, then the type view with groups (L2) renders
- [ ] Given the route `/dashboard/assets/[type]/[group]`, when accessed, then the group view with individual assets (L3) renders
- [ ] Given the routes `/dashboard/questionnaires` and `/dashboard/questionnaires/[id]`, when accessed, then the questionnaire list and questionnaire editor render respectively
- [ ] Given any route, when accessed directly (deep link), then the correct content loads without requiring navigation from the home page
- [ ] Given navigation from L1→L2→L3, when breadcrumbs are displayed, then they show the full path (e.g., "Portfolio > Acoes BR > Grupo 1") and clicking any parent segment navigates back to that level
- [ ] Given page transitions, when navigating, then `<ViewTransitions />` provides SPA-like transitions with no full-page reload
- [ ] Given a route with an invalid type slug (e.g., `/dashboard/assets/invalid-type`), when accessed, then a 404 page is displayed with a link back to `/dashboard/portfolio`
