# Story 4.3 — L3 Distribution (Asset-Level Rebalancing + FLOOR Logic)

**Phase:** 4 — Rebalancing Algorithm
**Status:** Draft
**Effort:** M
**Dependencies:** 4.2 (L2 must produce per-group amounts), 7.3 (score normalization for ideal % per asset), ADR-004 (Rebalancing Algorithm Architecture)

## Description

As a user,
I want the L3 cascade to distribute each group's allocated amount among individual assets based on their normalized scores, applying FLOOR for whole-share assets and fractional for ETFs,
so that I get an exact "buy list" showing ticker, quantity, and estimated cost per asset.

L3 uses normalized scores from questionnaires to determine each asset's ideal % within its group. It then calculates shares to buy using FLOOR (Math.floor) for stocks/FIIs and fractional for international ETFs.

[Source: PRD §3 Module 2, Feature F-002 AC5/AC6/AC7]

## Acceptance Criteria

- [ ] Given a group with 5 assets and normalized scores, when `distributeL3()` runs, then each asset receives an amount proportional to its normalized score
- [ ] Given an asset with `whole_shares = true` (stock/FII), when shares are calculated, then `Math.floor(amount / price)` is used (no fractional shares)
- [ ] Given an asset with `whole_shares = false` (ETF international), when shares are calculated, then fractional quantities are allowed
- [ ] Given the output, when displayed, then each asset shows: `ticker`, `shares_to_buy`, `estimated_cost_brl`
- [ ] Given an asset with `is_active = false` or `manual_override = true`, when L3 processes its group, then the asset is excluded from distribution and receives 0 shares
- [ ] Given FLOOR rounding, when computed, then the "remainder" (amount not spent due to rounding) is tracked per group
- [ ] Given the same inputs, when run twice, then outputs are identical (deterministic)
- [ ] Error: Given a group where all assets are inactive or overridden, when L3 runs, then the group's amount is returned as unallocated remainder
