# Story 12.2 — Wallet Management (Create, Rename, Delete)

**Epic:** 12 — Wallet UX
**Status:** Draft
**Effort:** S (2 points)
**Dependencies:** 12.1 (wallet selector, which provides the UI entry point)
**PRD Feature:** F-024

## Description

As a user,
I want to create new wallets, rename existing ones, and delete wallets I no longer need,
so that I can organize my investment portfolios as my needs evolve.

Wallet deletion requires a confirmation dialog because it cascade-deletes all associated data (asset types, groups, assets, scores, contributions).

[Source: PRD-v2 §3 Module 12, Feature F-024]

## Acceptance Criteria

- [ ] Given the "Nova carteira" action, when triggered, then a modal opens with a name input field (1-50 chars)
- [ ] Given a valid wallet name, when "Criar" is clicked, then the wallet is created and automatically becomes the active wallet
- [ ] Given an existing wallet, when a rename action is triggered, then an inline edit or modal allows changing the name (1-50 chars)
- [ ] Given a wallet, when the delete action is triggered, then a confirmation dialog warns: "Ao excluir esta carteira, todos os dados associados (classes, grupos, ativos, questionários) serão removidos permanentemente."
- [ ] Given confirmation of delete, when confirmed, then the wallet and all cascade data are removed, and the user is switched to the next available wallet (or redirected to onboarding if last wallet)
- [ ] Given the user has only one wallet, when they attempt to delete it, then the delete is allowed (results in onboarding flow per F-025)
- [ ] Given no wallet limit, when a user creates wallets, then there is no artificial limit on the number of wallets
