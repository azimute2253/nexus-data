# Story 3.5 — Manual Price Refresh Button

**Phase:** 3 — Price Engine
**Status:** Ready for Review
**Effort:** S
**Dependencies:** 3.4 (cron Edge Function must exist), 5.1 (dashboard page to host the button)

## Description

As a user,
I want to click an "Atualizar precos" button that forces an immediate price refresh,
so that I can get fresh prices on-demand before running a rebalancing calculation.

The button triggers the same Edge Function as the cron, but with a 1-minute debounce to prevent rate limit abuse.

[Source: PRD §3 Module 5, Feature F-007 AC5]

## Acceptance Criteria

- [ ] Given the dashboard page, when rendered, then an "Atualizar precos" button is visible
- [ ] Given the button is clicked, when the Edge Function completes, then all prices in `price_cache` are refreshed with new `fetched_at` timestamps
- [ ] Given the button was clicked less than 1 minute ago, when clicked again, then the request is debounced (no duplicate Edge Function invocation)
- [ ] Given the refresh is in progress, when the button is visible, then it displays a spinner icon and the button is disabled (non-clickable)
- [ ] Given prices are refreshed, when the dashboard re-renders, then updated prices and timestamps are displayed
- [ ] Error: Given the Edge Function fails, when the button handler catches the error, then an error message is shown to the user
