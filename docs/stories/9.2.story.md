# Story 9.2 — Visual Integration with azimute-blog Layout

**Phase:** 9 — Integration & Routes
**Status:** Draft
**Effort:** S
**Dependencies:** 9.1 (routes must exist), 1.3 (auth)

## Description

As a user,
I want the Nexus Data pages to visually integrate with the existing azimute-blog layout, navigation, and design system,
so that the portfolio dashboard feels like a native part of the site, not a bolted-on app.

[Source: PRD §3 Module 9, Feature F-011 AC4]

## Acceptance Criteria

- [ ] Given the dashboard pages, when rendered, then they use the same header, footer, and navigation sidebar as the azimute-blog members area
- [ ] Given the azimute-blog navigation, when inspected, then a "Portfolio" link/tab is present in the members area navigation
- [ ] Given the Nexus Data pages, when styled, then they use the same Tailwind theme tokens (colors, fonts, spacing) as the blog
- [ ] Given the portfolio dashboard, when accessed from the members area, then the transition preserves the same header/footer/nav and Cumulative Layout Shift (CLS) is < 0.1
- [ ] Given the shared layout, when Nexus Data components are added, then no blog-specific components are imported in Nexus Data code (ADR-001 Obligation 2)
- [ ] Given the shared layout fails to load (e.g., header/footer component throws), when the dashboard page is accessed, then a fallback layout renders with a minimal nav bar and an error message "Layout indisponivel — tente recarregar" instead of a blank page
