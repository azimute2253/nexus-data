# Story 8.2 — Mobile Rebalance Flow (2 Toques)

**Phase:** 8 — Mobile Responsiveness
**Status:** In Progress
**Effort:** S
**Dependencies:** 5.5 (rebalance calculator UI), 8.1 (mobile layout)

## Description

As a mobile user,
I want to complete the "digitar aporte → ver lista de compras" flow in 2 toques (taps),
so that I can quickly determine what to buy while in the corretora app.

The mobile flow: (1) tap the aporte input → (2) enter value and tap "Calcular" → result appears. The input and button must be prominently placed and thumb-friendly.

[Source: PRD §3 Module 7, Feature F-009 AC2]

## Acceptance Criteria

- [x] Given the mobile dashboard, when loaded, then the aporte input field and "Calcular" button are immediately visible (above the fold)
- [x] Given the mobile flow, when the user taps the input, types a value, and taps "Calcular", then the complete flow is achievable in 2 taps (input focus + calcular)
- [x] Given the result on mobile, when displayed, then the buy list renders in card format with font size >= 14px and line height >= 1.4
- [x] Given the result, when scrolled, then each asset card shows: ticker, shares, cost — no truncation
- [x] Given the aporte input, when displayed on mobile, then it uses `inputmode="decimal"` for the numeric keyboard
- [x] Error: Given the user enters a non-numeric value (e.g., "abc") or zero in the aporte input and taps "Calcular", when processed, then an inline validation error is displayed and no calculation is triggered
